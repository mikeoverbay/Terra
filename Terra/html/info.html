Terra! is intended to be used to plan
Clan War Battles for World of Tanks but also just to view the maps.

Moving around..

Left Mouse Button: Move camera around.
Right Mouse Button: Zoom in and out.
Center Mouse Button: Select Items. (Center clicking items in view will show information in the bottom text window.)
Shift + Left Mouse Button: Move look at position.
Click the Mini Map Rapid move to anywhere on the map.


You can rotate the turret on the selected tank using the mouse wheel.
You can change the gun elevation using the mouse wheel + spacebar.

Terra has undergone a complete rewrite of the rendering process.
Before, everything was done doing what is known as "Forward Rendering".
Now.. it is done using "Deferred Rendering". The advantage of this method is
the lighting caculations are done only on the pixels that are visible on the screen.
This saves a lot of time and speeds things up drastically. Also, you can have many
lights in the scene. As of now, there is only main light.. the Sun light position and
the added lights of the lamps. (My hack)
I added FXAA anti-aliasing to the final stage and the results are acceptable.
There are newer and better versions of FXAA coming out so in time this will be improved.
The down side to Deferred Rendering is the memory the buffers use. This is the first version
of the new rendering pipeline and will be improved as time allows.
Decals are now projected and use NO mesh what so ever to render them.
They are painted directly on the terrain and other objects. This is a farily complicated process because
they must be blended and the normal maps have to be blended as well.
This requires 2 full buffer reads per render loop.
My GTX 660 Ti has no problem handling this but slower, low memory cards
might choke on the bandwidth required.

When you first start up Terra!, you may need to set the path to the World of Tanks root folder.
You can do this under the File menu item. After you navigate to the folder "World of Tanks" and click OK, Terra! will close.
This is because many of the GUI items need to load in order for Terra! to run correctly. Terra! will remember this path so you wont need to set it every time you start Terra!

World of Tanks will need to ne installed on the system you run Terra! on.

Things you need to remember!!
The RED ring is always "Side One" or "I" and the GREEN ring is always side "II".
You can change the Mini Map size using "+" and "-".
You can also change the Mini Map tank icon sizes using Ctrl + "+" and Ctrl + "-".

There are a number of settings that control what's rendered and with what quality.
These are intended to make the rendering work 'good enough' on older systems.
There are a couple of settings like, showing the chuck boarders and map grid that is just for fun and not very useful.
Unless you tell Terra! to use billboard trees, it will use full 3D models of them extracted from the ctree files. This can hurt render time and slow things down.
Setting the tress to low quality will speed things up. Also, there is a setting to stop the loading of all the extra environment items such as curbs, light posts.. tables.. chairs.. anything that isn't part of the terrain or a building. This will reduce the load on the rendering engine and speed things up as well (More FPS is a good thing!)
NOTE: Changing the texture quality to low, only effects maps loaded after the change in that setting.
The Textures will be resized when loaded to a maximum of 256 pixels wide. This can save render time as well as a lot of memory.

The current Sector is displayed at the bottom left of the rendering window.

Team Layout:
Clicking on the 'Team Layout' button will open the team layout panel.
There are 2 teams.. as in the game.. Red and Green.
In order to deploy a tank, you must first assign a position to one of the team lists.
Once you have selected an open position, decide on what tank you want and then click it on the right side.
This will move that tank to that position in the team's list you selected first. You can replace any tank in the team lists at any time.
There are a number of tanks from each nation you can chose from. I selected some of the most common tanks for CWs and a few other lower tiers for good measure.
NOTE: The tank will be placed where the cursor is, facing the same direction you are looking.
By position the cursor and camera, you can place tanks with out having to rotate them after but in most cases, you will need to.
If you do need to move a tank.. You can select a tank two ways.. either center click it with the mouse or select it on the layout list. The team layout list MUST be visible to select tanks.
Selecting tanks that are already on the layout list will place the cursor at that tanks location. You can quickly move tanks anywhere on the map by selecting one and than clicking the minimap. This will move that tank to where you clicked on the map.
A selected tank will be Yellow in color . Once a tank is selected.. there are a few ways to move it.
You can press and hold the T key and move the tank around by dragging it with the mouse.. You can also rotate the tank by holding down the R key and dragging the mouse... There also is the option of using the mini map to move the tank anywhere on the map.

So the basic process of creating a team goes like this.
1. Decide where you want your tanks.
2. Pick a 'open' spot in that team's list and then..
3. Pick a tank from the list of tanks.
4. Deselect that tank by middle mouse clicking anywhere but on a tank.
5. Move the cursor to where you want the next tank (SHIFT + MOUSE DRAG) and pick the next open team position, select the tank for it and .. keep repeating this until all your tanks are placed where you like them. If you don't like where a tank is, move it by selecting it and moving and rotating it to how you like. You can reset the layout by clicking.. Oddly enough.. the Reset Layout menu item.. This will ask you if you are 100% sure as it can NOT be undone. Once you have your teams the way you like, Save it.. I would use names like "Air Field-Plan A" or "Fjords Setup 1" or.. you could even include a date.
NOTE: The engine does not care where you place a tank.. Be aware that in the game, there are places tanks can NEVER get to so placing tanks on top of a mountain may look cool, in the real world of tanks, a tank could never make it there.

WANs, LANs and Sessions.
All of this has proven to be problematic and near useless.
I should remove it all at some point. You are still welcome to try it out.
Terra! has a build in server which allows your Clan mates to participate in the design process.
The Server and Client base code is from a man with the name of Lidgren. It is a fantastic DLL which makes multi-client supported servers much easier to implement.
To host a session, click 'Host a Session' from the Session menu item.
This will open up the server controller.
Although this window is small, there is a tremendous amount of code behind it.
WAN = Wide Area Network.
LAN = Local Area Network.
On this control, there are a couple of buttons that are links to get help you may need. Port Forwarding and also an easy way to find your 'out side IP' or WAN.
If you don't know what Port Forwarding is, you need to read up at the link that button will take you to. Terra! uses port number 1956.
Basically, port forwarding maps a port to a particular IP on your LAN. It is usually best to setup a Static IP on the computer you will host Terra! on or any other application that has a port mapped to that machine.
You will need to know your WAN in order for others to connect to Terra!.When people connect to you, they will enter your WAN IP. Port forwarding will pass the port to your computer and the connection will be made.
LAN addresses always look like this: 192.168.xx.xx. Don't get this confused with WAN addresses. The Host server will listen on the IP that says 'Your WAN IP'. If this address is not correct, the host will be listening but not on your actual IP and so.. it wont be able to detect any in coming connection request.
Terra! will remember what you put in the setup boxes so you wont need to do it every time.. However.. If your IP changes for what ever reason, Terra! has no way of knowing this and you will need to update the IP box. You will also notice there is a box for a password. The Server will reject any connection attempt that does not match this password. Keep the password simple.. Its just to stop anyone from accidentally logging in using port 1956. You will also see there is a Diagnostics button. This will show some info while the server is running such as.. the ping for each client that has connected and any strange errors that might show up. Once you have everything setup.. Click the Start Button....
The server will Minimize to the task bar and the client chat window will appear. If you are connecting to some one else's computer, Put the IP of that computer in the 'Host to connect to' box. If you are hosting a session.. put in this IP: 127.0.0.1 <-- this is the loop back address and will save network bandwidth. You can of course use the WAN or LAN IP for the local server though I don't recommend it. If you are joining a session.. you need the IP of the computer you are connecting to and the password that was assigned.

You can setup a remote server by downloading and installing the server on your VPS or any computer you chose. Setup its IP.. Password and start it.
To connect to the remote server as the host, use "Join Remote as Host" from the Session drop down menu. You may need to setup port forwarding if this server is part of your LAN.

If you are the one Hosting a Session, your name will be Red in the client list.. Other wise, it will be Black.
Who ever is the current drive will be displayed in Yellow.

If you try to connect to remote server as the host and the there is already someone hosting on that server, you will be disconnected and a message will show up in the Diag. area.

Passing the Driving control.
One of the things I designed in to Terra! is the ability to pass the driving control to some one else.
How this works is, If you are the host OR the current driver, you can reassign who has driving control by Clicking on one of the client names. The controlling person's name will be yellow. Drivers can move around on the map.. move the tanks and change the tank layout. Only the host can force a map reload or force a team layout reset.
Only the Host can pass the control to another but if he/she does,.. that person can pass the control on to some one else. The Host ALWAYS has the ability to take the control away or reassign it to some one else. Once the control is passed, that person has no more control over driving.
In the event that the current driver disconnects, the Host will become the driver automatically. If the host times out or disconnects.. If the server was local on the host's computer, the session is over because the server was started by the host. The host can reconnect using the "Join Remote as Host" if the server is remote.

The chat window is the same as any other you will find.. Type.. press enter to send. You can use SHIFT+ ENTER to break lines before sending.
The Server buffers 10 chat messages so when you connect, you can see the conversation that has be taking place.
